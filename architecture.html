Xively Architecture

IoT Connected Product Architecture

Below is the architecture for the Xively Product Simulator leveraging Heroku and Salesforce to provide a reference implementation of a connected product with an end user app and service automation.

This product simulator and reference app architecture can be used to developed and adapted to build a customized, commercial IoT implementation.

Deploy Xively

Product Simulator

Model a connected product, the data associated with it, and test new features and usage models across business systems. The simulator is customizable to model the specific data for any product and updates to the model are propagated across Salesforce, Heroku and Xively.



Device Agent Software

Xively provides a client library for use on devices. The library is designed for security and scalability to support resource-constrained device environments. It incorporates a variety of microcontroller families and includes features like remote administration.



Device Management App

Xively provides an application to see product data, manage deployed devices, and create a unified view of connected products.

Service Cloud Integration

Reference app links live contextual data from the Xively Product Simulator into Service Cloud via a Device Bridge. The Salesforce Xively Reference App can be deployed here on AppExchange.

End User App

Reference app providing an example end user view of a connected product based on the Xively Product Simulator. The reference app can be leveraged to build rich and engaging end user commercial grade app.



Heroku Connect

Seamlessly connect Xively Product Simulator to Salesforce and create cases or opportunities through Heroku Connect.
